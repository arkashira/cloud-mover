# Customer Journey Map – cloud‑mover

| Phase   | Trigger Event | Friction Points | User Emotions | Opportunities to Delight | Key Metric |
|---------|----------------|-----------------|---------------|--------------------------|------------|
| **Aware** | • A major cloud provider (e.g., AWS, GCP, Azure) announces a service outage or deprecation.<br>• DevOps engineer reads a blog post about multi‑cloud migration challenges. | • Information overload: many migration tools, unclear value proposition.<br>• Lack of concrete pricing or ROI data. | Curious → Concerned | • Publish a **quick‑start video** (2‑min) showing a 5‑minute switch from AWS to DigitalOcean using cloud‑mover.<br>• Offer a **live demo** in a 15‑min webinar with real‑time migration. | **Website visits from target orgs** (≥ 10 k/month) |
| **Consider** | • Team evaluates alternatives after the outage.<br>• Product manager searches “platform‑agnostic deployment manager”. | • Integration complexity with existing CI/CD pipelines.<br>• Security & compliance concerns (data residency, IAM).<br>• Uncertainty about vendor lock‑in. | Skeptical → Hopeful | • Provide a **sandbox environment** (free tier) with 30‑day trial and pre‑configured connectors.<br>• Publish **case studies** (e.g., 3‑month migration from AWS to Azure with zero downtime).<br>• Offer a **security white‑paper** detailing encryption, audit logs, and compliance certifications. | **Free trial sign‑ups** (≥ 200 per month) |
| **Try** | • DevOps team installs cloud‑mover CLI and runs a pilot migration. | • Installation hiccups (Docker vs. native binaries).<br>• Documentation gaps for custom resource definitions.<br>• Limited error‑handling messages. | Frustrated → Curious | • Add a **step‑by‑step wizard** in the CLI that auto‑detects current infra and suggests target platforms.<br>• Implement **real‑time telemetry** in the UI to show migration progress and rollback points.<br>• Provide a **community Slack channel** with live support. | **Time to first successful migration** (≤ 30 min) |
| **Adopt** | • Migration completes with minimal downtime.<br>• Team sees cost savings and flexibility. | • Scaling to production workloads.<br>• Managing multi‑cloud policies (cost, security).<br>• Need for ongoing support and updates. | Confident → Loyal | • Offer a **managed service tier** (SLA, 24/7 support).<br>• Introduce **policy‑as‑code** templates for cost‑control and compliance.<br>• Integrate with **monitoring dashboards** (Prometheus, Grafana). | **Monthly recurring revenue (MRR)** from paid plans (≥ $5k/month) |
| **Expand** | • Company expands to new regions or adds new cloud providers (e.g., Alibaba Cloud, Oracle). | • Vendor‑specific quirks, new API versions.<br>• Need for advanced automation (CI/CD pipelines, GitOps). | Excited → Advocate | • Release **plugin marketplace** for community‑built connectors.<br>• Provide **GitOps‑ready Helm charts** and Terraform modules.<br>• Run **customer success workshops** to uncover upsell opportunities (e.g., multi‑region disaster recovery). | **Net Promoter Score (NPS)** (target ≥ 70) |
